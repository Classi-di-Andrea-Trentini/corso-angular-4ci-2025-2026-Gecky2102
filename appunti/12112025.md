### Regex
Le espressioni regolari (regex) sono sequenze di caratteri che formano un modello di ricerca. Vengono utilizzate per trovare e manipolare stringhe di testo in base a determinati schemi.
Esempio di regex per validare una classe scolastica:
```TypeScript
const regex = /^[1-5][ABCDEQ][iame]?$/;
```
- `^` indica l'inizio della stringa.
- `[1-5]` indica che il primo carattere deve essere un numero da 1 a 5.
- `[ABCDEQ]` indica che il secondo carattere deve essere una lettera tra A, B, C, D, E o Q.
- `[iame]` indica che il terzo carattere deve essere una lettera tra i, a, m o e.
- `?` indica che il terzo carattere è opzionale.
- `$` indica la fine della stringa.

### Gestione Errori
In TypeScript, è possibile gestire gli errori utilizzando i blocchi `try`, `catch` e `finally`. Esempio:
```TypeScript
try {
    // Codice che potrebbe generare un errore
} catch (error) {
    // Gestione dell'errore
} finally {
    // Codice che viene eseguito sempre, indipendentemente dal fatto che si sia verificato un errore o meno
}
```
Il throw viene utilizzato per generare un errore personalizzato. Esempio:
```TypeScript
if (value.length === 0) {
    throw new Error('Valore non valido!');
}
```
