### Classi e Interfacce in TypeScript

Perchè programmare in classi?
 - Le classi permettono di creare oggetti con proprietà e metodi, facilitando la programmazione orientata agli oggetti.
 - Possono essere riutilizzate e estese, migliorando la manutenibilità del codice.

Le classi si generano con il comando:
```TypeScript
ng generate class components/nome-classe
```
Questo comando creerà un file `nome-classe.ts` e un `nome-classe.spec.ts` nella cartella `components`.

Esempio di classe studente con costruttore e getter:
```TypeScript
export class Studente {
  private _nome: string;
  private _cognome: string;
  private _eta: number;

  constructor(nome: string, cognome: string, eta: number) {
    this._nome = nome;  
    this._cognome = cognome;
    this._eta = eta;
  }

  public get nome(): string {
    return this._nome;
  }

  public get cognome(): string {
    return this._cognome;
  }

  public get eta(): number {
    return this._eta;
  }

  public getNomeCompleto(): string {
    return this._nome + ' ' + this._cognome;
  }

  public set nome(nome: string) {
    this._nome = nome;
  }
  
  public set cognome(cognome: string) {
    this._cognome = cognome;
  }

  public set eta(eta: number) {
    this._eta = eta;
  }
}
```

Nel costruttore per ereditare una classe si usa la parola chiave `extends`:
```TypeScript
export class StudenteUniversitario extends Studente {
  private _classe: string;
    constructor(nome: string, cognome: string, eta: number, classe: string) {
        super(nome, cognome, eta);
        this._classe = classe;
    }
    public get classe(): string {
        return this._classe;
    }
    public set classe(classe: string) {
        this._classe = classe;
    }
}
```
