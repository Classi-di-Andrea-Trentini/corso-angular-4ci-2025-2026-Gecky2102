### Undefined e Null in TypeScript
In TypeScript, `undefined` e `null` sono due tipi di valori che rappresentano l'assenza di un valore, ma hanno significati leggermente diversi:
- `undefined`: Indica che una variabile è stata dichiarata ma non è stata ancora assegnata a un valore. È il valore predefinito per le variabili non inizializzate.
- `null`: Rappresenta intenzionalmente l'assenza di un valore.
- Esempio di utilizzo di `undefined` e `null` in TypeScript:
```TypeScript
let variabileNonInizializzata: string | undefined;
console.log(variabileNonInizializzata); // Output: undefined 
let variabileConNull: string | null = null;
console.log(variabileConNull); // Output: null
```

In TypeScript si può utilizzare un operatore chiamato operatore di navigazione sicura (?.) per accedere in modo sicuro alle proprietà di un oggetto che potrebbe essere `null` o `undefined`. Questo operatore evita errori di runtime quando si tenta di accedere a una proprietà di un oggetto che non esiste.
Esempio di utilizzo dell'operatore di navigazione sicura in una String Interpolation:
```TypeScript
let persona: { nome: string; indirizzo?: { citta: string } } = {
  nome: "Mario Rossi",
};
console.log(`Città: ${persona.indirizzo?.citta}`); // Output: Città: undefined
```
In questo esempio, l'operatore `?.` viene utilizzato per accedere alla proprietà `citta` dell'oggetto `indirizzo`. Poiché `indirizzo` è `undefined`, l'operatore di navigazione sicura impedisce un errore di runtime e restituisce `undefined` invece di tentare di accedere a una proprietà di un oggetto non definito.
