### API RESTful in Angular
- Introduzione alle API RESTful e come interagire con esse utilizzando Angular.
- API Significa Application Programming Interface, un insieme di regole che permettono a due applicazioni di comunicare tra loro.
- Le richieste HTTP o HTTPS sono il metodo più comune per interagire con le API RESTful.
- HTTP Significa HyperText Transfer Protocol, è il protocollo di comunicazione utilizzato per trasferire dati sul web.
- HTTPS è la versione sicura di HTTP, che utilizza la crittografia per proteggere i dati durante il trasferimento.
- Come tutti i protocolli HTTP prevede una serie di verbi. Ad esempio quando facciamo una ricerca sul nostro browser, stiamo facendo una richiesta HTTP di tipo GET.
- URI (Uniform Resource Identifier) è l'indirizzo che identifica in modo univoco una risorsa su internet.
- Altri verbi HTTP comuni includono POST (per creare nuove risorse), PUT (per aggiornare risorse esistenti) e DELETE (per eliminare risorse).
- Questi verbi vengono utilizzati quando ci rivolgiamo ad un'API RESTful per eseguire operazioni sulle risorse.
- RESTful significa che l'API segue i principi del REST (Representational State Transfer), un'architettura che utilizza le risorse e i verbi HTTP per creare un'interfaccia semplice e scalabile.
- Quando usiamo ad esempio nel nostro caso l'api di Spotify, useremo i verbi GET e POST. Per entrambi devo specificare un URI, ovvero l'indirizzo della risorsa a cui voglio accedere e che voglio mi restituisca.
- Il verbo POST permetti di inviare prima una riga vuota e poi un body, ovvero un insieme di dati che vogliamo inviare al server.
- I dati come username e password se utiliazziamo HTTP saranno inviati in chiaro, mentre con HTTPS saranno criptati.
- Un messaggio prevede il verbo (GET, POST, PUT, DELETE), l'URI, le intestazioni (headers) e il corpo (body).
- L'estensione REST CLIENT ci permette di comporre a mano un messaggio HTTP strutturato in questo modo. [ESEMPIO RICHIESTA](../spotify/test.http)
- Bearer Token è un tipo di token di autenticazione che viene utilizzato per autorizzare l'accesso alle risorse protette su un server.
- Un sito può avere la necessità di memorizzare delle informazioni sul client, ad esempio per tenere traccia della sessione di un utente. Per fare questo utilizza i cookie (client) o le sessioni (server).
